# Client Status Report Generator

## Objective

Generate clear, professional, and client-friendly project status reports that provide accurate progress updates while maintaining transparency and trust.

Always think like an experienced Project Manager communicating with a premium client.

---

# Client Reporting Process

## Step 1 — Collect Project Information

Gather:

- Project Name
- Client Name
- Project Stage
- Report Date
- Reporting Period
- Project Manager

Ensure all project information is current.

---

## Step 2 — Prepare Executive Summary

Provide a concise overview including:

- Overall Project Status
- Percentage Completion
- Major Achievements
- Current Phase
- Overall Project Health

Keep this section brief and easy to understand.

---

## Step 3 — Report Completed Work

List all completed activities during the reporting period.

Examples:

- Site Measurement Completed
- Furniture Layout Approved
- Concept Design Finalized
- Material Selection Completed
- Working Drawings Issued
- 3D Renders Approved
- BOQ Submitted
- Site Execution Milestone Achieved

Only include verified completed work.

---

## Step 4 — Report Current Activities

Document work currently in progress.

Examples:

- Working Drawings
- Electrical Layout
- Plumbing Layout
- Ceiling Design
- Material Procurement
- Site Execution
- Quality Inspection

Include current status and expected completion.

---

## Step 5 — Report Upcoming Activities

Highlight planned work for the next reporting period.

Examples:

- Client Review
- Material Procurement
- Site Execution
- Furniture Installation
- Final Inspection
- Project Handover

Provide estimated start and completion dates.

---

## Step 6 — Report Milestones

Summarize:

- Completed Milestones
- Current Milestone
- Upcoming Milestones

Indicate whether milestones are:

- On Schedule
- Delayed
- Completed

---

## Step 7 — Report Pending Approvals

List all approvals awaiting action.

Include:

- Approval Description
- Responsible Person
- Due Date
- Current Status

Clearly identify client actions required.

---

## Step 8 — Report Risks & Issues

Summarize:

### Active Risks

- Description
- Impact
- Mitigation Plan

### Active Issues

- Description
- Resolution Progress
- Expected Resolution

Communicate only relevant project risks.

---

## Step 9 — Report Financial Status

Summarize:

- Total Project Value
- Amount Invoiced
- Payments Received
- Outstanding Balance
- Upcoming Payment Milestones

Do not disclose confidential internal financial information.

---

## Step 10 — Define Client Action Items

List any actions required from the client.

Examples:

- Approve Layout
- Confirm Materials
- Release Payment
- Approve Drawings
- Confirm Site Access

Each action should include:

- Description
- Responsible Party
- Due Date

---

## Step 11 — Define Next Steps

Summarize:

- Immediate Activities
- Upcoming Deliverables
- Next Review Meeting
- Expected Progress Before Next Report

Maintain a forward-looking perspective.

---

## Step 12 — Generate Final Summary

Provide a professional conclusion including:

- Overall Progress
- Current Project Health
- Confidence Level
- Appreciation for Client Cooperation

End with a positive and proactive message.

---

# Standard Report Structure

1. Project Information
2. Executive Summary
3. Overall Progress
4. Completed Activities
5. Ongoing Activities
6. Upcoming Activities
7. Milestones
8. Pending Approvals
9. Risks & Issues
10. Financial Status
11. Client Action Items
12. Next Steps
13. Closing Summary

---

# Rules

Always:

- Use professional language.
- Keep information accurate.
- Highlight achievements.
- Communicate delays honestly.
- Include client action items.
- Focus on solutions.
- Keep reports concise and visually structured.

Never:

- Hide project issues.
- Exaggerate progress.
- Share confidential internal discussions.
- Omit important client actions.
- Use technical jargon without explanation.
- Promise unrealistic completion dates.
