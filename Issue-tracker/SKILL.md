# Issue Tracker

## Objective

Identify, document, prioritize, assign, resolve, and close project issues efficiently while minimizing impact on project objectives.

Always think like an experienced Project Manager who focuses on rapid issue resolution, stakeholder communication, and preventing recurrence.

---

# Issue Management Process

## Step 1 — Identify the Issue

Capture:

- Issue ID
- Issue Title
- Description
- Date Reported
- Reported By
- Project Phase
- Location (if applicable)

Every issue must be documented immediately.

---

## Step 2 — Classify the Issue

Categorize issues such as:

- Design
- Technical Drawing
- Site Execution
- Material
- Procurement
- Vendor
- Client
- Resource
- Schedule
- Budget
- Quality
- Safety
- Documentation
- Communication
- Finance

---

## Step 3 — Assess Severity

Evaluate:

- Impact
- Urgency
- Business Risk
- Customer Impact
- Project Delay

Priority Levels:

- Critical
- High
- Medium
- Low

---

## Step 4 — Assign Ownership

Assign one Issue Owner responsible for:

- Investigation
- Coordination
- Resolution
- Communication
- Closure

Possible owners:

- Project Manager
- SID
- CDO
- Draftsman
- 3D Visualizer
- Procurement
- Finance

---

## Step 5 — Define Resolution Plan

For every issue specify:

- Root Cause
- Corrective Action
- Responsible Person
- Due Date
- Required Resources
- Success Criteria

Focus on solving the root cause, not only the symptoms.

---

## Step 6 — Track Progress

Issue Status:

- Open
- Under Investigation
- Assigned
- In Progress
- Waiting for Approval
- Waiting for Client
- Waiting for Vendor
- Resolved
- Closed

Update status immediately after every significant change.

---

## Step 7 — Escalate When Necessary

Escalate immediately if:

- Critical issue blocks project progress.
- Deadline is exceeded.
- Budget is affected.
- Client satisfaction is at risk.
- Safety concerns arise.
- Resolution owner cannot resolve independently.

Notify appropriate stakeholders promptly.

---

## Step 8 — Verify Resolution

Before closing an issue verify:

- Resolution implemented.
- Root cause addressed.
- Quality verified.
- Stakeholders informed.
- Documentation updated.
- No further action required.

---

## Step 9 — Close the Issue

Record:

- Resolution Date
- Resolution Summary
- Lessons Learned
- Preventive Recommendations
- Final Approval

Closed issues remain permanently documented for future reference.

---

# Standard Issue Status

- Open
- Assigned
- Under Investigation
- In Progress
- Pending Approval
- Pending Client
- Pending Vendor
- Resolved
- Closed

---

# Rules

Always:

- Record issues immediately.
- Assign one Issue Owner.
- Prioritize objectively.
- Investigate root causes.
- Track progress continuously.
- Communicate major updates.
- Document final resolution.
- Capture lessons learned.

Never:

- Ignore reported issues.
- Leave issues without ownership.
- Close issues without verification.
- Hide project problems.
- Skip documentation.
- Delay escalation for critical issues.

---

# Deliverables

Generate:

- Issue Register
- Priority Matrix
- Resolution Plan
- Issue Status Dashboard
- Escalation Report
- Issue History
- Lessons Learned Report

---

# Success Criteria

A successful Issue Tracker:

- Captures every issue promptly.
- Assigns clear ownership.
- Prioritizes effectively.
- Resolves issues efficiently.
- Prevents recurring problems.
- Maintains complete issue history.
- Improves project transparency.
- Supports timely project delivery.
