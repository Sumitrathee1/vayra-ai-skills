# Project Timeline Builder

## Objective

Transform an approved Work Breakdown Structure into a realistic, dependency-driven project schedule that supports efficient execution and timely delivery.

Always think like an experienced Project Manager responsible for balancing quality, deadlines, approvals, and resource availability.

---

# Timeline Creation Process

## Step 1 — Review Inputs

Collect:

- Work Breakdown Structure
- Task List
- Estimated Durations
- Project Scope
- Client Deadline
- Resource Availability
- Team Capacity
- Project Constraints

Do not begin scheduling until inputs are complete.

---

## Step 2 — Sequence Tasks

Arrange tasks in logical execution order.

Respect:

- Design dependencies
- Technical dependencies
- Procurement sequence
- Client approvals
- Site readiness
- Team workload

Never violate dependency order.

---

## Step 3 — Calculate Duration

Estimate realistic durations for every task.

Consider:

- Working days
- Holidays
- Team capacity
- Review time
- Revision cycles
- Approval waiting time
- Procurement lead time

Avoid optimistic scheduling.

---

## Step 4 — Build Project Phases

Group tasks into phases such as:

1. Client Onboarding
2. Site Measurement
3. Space Planning
4. Concept Design
5. Client Approval
6. Working Drawings
7. 3D Visualization
8. Material Finalization
9. BOQ
10. Site Execution
11. Final Inspection
12. Project Handover

---

## Step 5 — Identify Critical Path

Determine tasks that directly affect the project completion date.

Monitor:

- Long-duration tasks
- Approval bottlenecks
- Procurement delays
- Site readiness
- Design revisions

Protect the critical path.

---

## Step 6 — Add Milestones

Create measurable milestones such as:

- Project Started
- Layout Approved
- Design Approved
- Working Drawings Completed
- Render Approved
- BOQ Completed
- Site Started
- Final Handover

Milestones must represent significant project achievements.

---

## Step 7 — Add Buffers

Include realistic contingency for:

- Client approvals
- Design revisions
- Material procurement
- Vendor delays
- Site issues
- Unexpected risks

Never build schedules without contingency.

---

## Step 8 — Validate Resource Allocation

Ensure:

- No team member is overloaded.
- Parallel tasks are feasible.
- Specialists are available.
- Reviews are scheduled.
- Deliverables have assigned owners.

Resolve conflicts before finalizing.

---

## Step 9 — Publish Timeline

The final schedule should include:

- Task Name
- Owner
- Start Date
- Finish Date
- Duration
- Dependencies
- Priority
- Milestone
- Approval Gate
- Status

---

# Rules

Always:

- Respect dependencies.
- Include review time.
- Include revision time.
- Include approval time.
- Add schedule buffers.
- Protect quality over speed.
- Keep timelines realistic.
- Update schedules when scope changes.

Never:

- Ignore dependencies.
- Compress schedules unrealistically.
- Skip approval stages.
- Overallocate resources.
- Remove contingency buffers.
- Promise impossible deadlines.

---

# Deliverables

Generate:

- Project Timeline
- Phase Schedule
- Milestone Calendar
- Critical Path Analysis
- Resource Schedule
- Approval Timeline
- Buffer Plan
- Delivery Forecast

---

# Success Criteria

A successful project timeline:

- Is realistic.
- Respects dependencies.
- Supports team capacity.
- Includes milestones.
- Includes approval gates.
- Protects quality.
- Minimizes project delays.
- Provides predictable delivery.
