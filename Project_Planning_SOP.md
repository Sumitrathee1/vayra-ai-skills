# Project Planning SOP

## Purpose

Establish a complete execution plan before any project begins. This SOP ensures every interior design project at Vayra Atelier starts with a clearly defined scope, timeline, responsibilities, dependencies, risks, and approval process.

---

# Objective

Create an executable project roadmap that:

- Aligns all stakeholders.
- Prevents scope confusion.
- Defines ownership.
- Improves delivery predictability.
- Reduces project risk.
- Ensures every department begins work with complete clarity.

---

# When to Use

Use this SOP whenever:

- A new project is awarded.
- A new phase begins.
- Major scope changes occur.
- Client requirements change.
- Timeline must be rebuilt.
- Project restarts after long delays.

---

# Required Inputs

Before planning, collect:

- Client Brief
- Approved Scope
- Site Information
- Area Statement
- Budget
- Timeline Expectations
- Design Requirements
- Team Availability
- Client Priorities
- Approval Requirements
- Existing Drawings (if any)
- Site Constraints

If any required information is missing, request clarification before creating the execution plan.

---

# Planning Workflow

## Step 1 — Understand the Project

Study:

- Client objectives
- Business goals
- Functional requirements
- Design expectations
- Budget limitations
- Completion deadline
- Special requirements
- Site constraints

Never assume missing information.

---

## Step 2 — Define Deliverables

Break the project into deliverables.

Example:

- Site Measurement
- Furniture Layout
- Concept Design
- Mood Board
- Material Selection
- 3D Visualization
- Working Drawings
- BOQ
- Site Coordination
- Final Handover

Every deliverable must have a clear completion definition.

---

## Step 3 — Break Work into Phases

Typical phases include:

1. Client Onboarding
2. Requirement Collection
3. Site Measurement
4. Space Planning
5. Design Development
6. Client Approval
7. Working Drawings
8. 3D Visualization
9. Material Finalization
10. BOQ Preparation
11. Execution Support
12. Site Supervision
13. Snag Resolution
14. Final Handover

Adjust phases according to project scope.

---

## Step 4 — Create Task Breakdown

For every phase define:

- Tasks
- Subtasks
- Expected output
- Dependencies
- Responsible owner
- Estimated duration

Every task must have only one primary owner.

---

## Step 5 — Assign Responsibilities

Assign work using ownership principles.

For every task specify:

- Responsible
- Accountable
- Consulted
- Informed

Never leave ownership undefined.

---

## Step 6 — Build Timeline

Estimate:

- Start Date
- End Date
- Duration
- Buffer Time
- Critical Path
- Approval Waiting Time

Avoid overlapping dependent tasks.

---

## Step 7 — Identify Dependencies

Document:

- Internal dependencies
- Client approvals
- Vendor dependencies
- Material availability
- Site readiness
- Drawing completion
- Visualization completion

Every dependency must have an owner.

---

## Step 8 — Identify Risks

Review risks related to:

- Client approvals
- Budget
- Procurement
- Timeline
- Site conditions
- Design revisions
- Material availability
- Vendor delays
- Documentation
- Team capacity

Record mitigation actions for every major risk.

---

## Step 9 — Define Approval Gates

Identify approval checkpoints such as:

- Furniture Layout Approval
- Concept Approval
- Material Approval
- Render Approval
- Working Drawing Approval
- BOQ Approval
- Final Completion Approval

No phase begins without required approval.

---

## Step 10 — Publish Project Plan

The final execution plan should include:

- Project Summary
- Scope
- Deliverables
- Timeline
- Milestones
- Owners
- Dependencies
- Risks
- Approval Matrix
- Communication Plan

---

# Planning Principles

Always:

- Plan before execution.
- Build realistic schedules.
- Include review time.
- Include revision time.
- Include client approvals.
- Consider procurement lead times.
- Protect quality over speed.
- Document assumptions.

Never:

- Skip planning.
- Hide risks.
- Ignore dependencies.
- Promise unrealistic deadlines.
- Begin work without ownership.
- Create plans without client scope.

- Eliminates ambiguity.
- Clearly defines ownership.

- Supports every department.
- Minimizes project risk.
- Enables predictable delivery.
- Improves client confidence.
- Becomes the single source of execution truth.
