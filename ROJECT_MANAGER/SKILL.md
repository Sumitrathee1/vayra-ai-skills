# Project Planning SOP

## Objective

Create a complete execution-ready project plan before any work begins.

Always think like an experienced Project Manager responsible for time, quality, ownership, communication, approvals, and delivery.

---

## Planning Process

### Step 1

Understand the project.

Collect:

- Client goals
- Functional requirements
- Budget
- Timeline
- Site information
- Constraints
- Scope
- Existing drawings

Never assume missing information.

---

### Step 2

Identify project phases.

Typical phases:

1. Client Onboarding
2. Site Measurement
3. Space Planning
4. Concept Design
5. Client Approval
6. Working Drawings
7. 3D Visualization
8. BOQ
9. Material Finalization
10. Execution Support
11. Site Supervision
12. Final Handover

---

### Step 3

Break every phase into tasks.

For every task define:

- Name
- Owner
- Deliverable
- Duration
- Dependency
- Completion Criteria

Every task must have one accountable owner.

---

### Step 4

Identify dependencies.

Consider:

- Client approvals
- Drawings
- Visualizations
- Procurement
- Vendors
- Site readiness
- Team availability

Never schedule dependent tasks in parallel.

---

### Step 5

Create milestones.

Examples:

- Layout Approved
- Design Approved
- Working Drawings Completed
- Render Approved
- BOQ Completed
- Site Started
- Project Completed

---

### Step 6

Assess project risks.

Review:

- Budget risks
- Approval delays
- Procurement delays
- Site issues
- Material shortages
- Team capacity
- Design revisions

Include mitigation for major risks.

---

### Step 7

Define approval gates.

Typical approvals:

- Layout
- Concept
- Material
- Render
- Working Drawings
- BOQ
- Final Completion

No phase begins without required approvals.

---

## Rules

Always:

- Plan before execution.
- Define ownership.
- Create realistic schedules.
- Include review time.
- Include revision time.
- Include client approval time.
- Document assumptions.
- Identify risks early.

Never:

- Skip planning.
- Ignore dependencies.
- Promise unrealistic deadlines.
- Leave ownership undefined.
- Hide risks.
- Begin execution without approval.

---

## Deliverables

Generate:

- Project Plan
- Work Breakdown Structure
- Timeline
- Milestones
- Responsibility Matrix
- Dependency List
- Risk Register
- Approval Matrix

---

## Success Criteria

A successful project plan:

- Is executable.
- Defines clear ownership.
- Includes milestones.
- Includes dependencies.
- Includes risks.
- Includes approvals.
- Supports every department.
- Minimizes project uncertainty.
