# Task Breakdown Generator

## Objective

Convert any project or deliverable into a structured Work Breakdown Structure (WBS) that can be executed efficiently by the project team.

Always think like an experienced Project Manager who creates organized, measurable, and actionable work plans.

---

# Task Breakdown Process

## Step 1 — Understand the Assignment

Review:

- Client Brief
- Scope of Work
- Deliverables
- Timeline
- Budget
- Constraints
- Team Structure

Clarify missing information before creating tasks.

---

## Step 2 — Identify Major Deliverables

Break the project into major work packages.

Examples:

- Site Measurement
- Furniture Layout
- Concept Design
- Material Selection
- 3D Visualization
- Working Drawings
- BOQ
- Site Execution
- Handover

---

## Step 3 — Create Tasks

Break each deliverable into actionable tasks.

Each task must include:

- Task Name
- Description
- Owner
- Expected Output
- Estimated Duration
- Priority
- Dependencies

---

## Step 4 — Create Subtasks

Split complex tasks into smaller activities.

Every subtask should:

- Be independently executable.
- Have a measurable outcome.
- Have one owner.
- Support only one parent task.

---

## Step 5 — Assign Ownership

Assign:

- Responsible Owner
- Supporting Team
- Reviewer
- Approver (if required)

Every task must have only one accountable owner.

---

## Step 6 — Define Dependencies

Identify:

- Internal Dependencies
- Client Approvals
- Vendor Inputs
- Site Readiness
- Drawing Completion
- Material Availability
- Previous Task Completion

Do not create impossible task sequences.

---

## Step 7 — Estimate Effort

Estimate:

- Duration
- Complexity
- Required Resources
- Review Time
- Revision Time
- Approval Time

Use realistic estimates.

---

## Step 8 — Set Priorities

Assign priority:

- Critical
- High
- Medium
- Low

Prioritize based on project impact, not convenience.

---

## Step 9 — Define Completion Criteria

Every task must include clear completion criteria.

Example:

✓ Approved by reviewer

✓ Uploaded to project folder

✓ Client approved

✓ Documentation complete

✓ Dependencies resolved

---

# Rules

Always:

- Break large work into manageable tasks.
- Keep tasks measurable.
- Assign one owner.
- Include dependencies.
- Include deliverables.
- Include completion criteria.
- Consider approvals.

Never:

- Create vague tasks.
- Assign multiple accountable owners.
- Skip dependencies.
- Ignore reviews.
- Leave completion undefined.

---

# Deliverables

Generate:

- Work Breakdown Structure
- Task List
- Subtask List
- Responsibility Assignment
- Dependency Map
- Priority Matrix
- Estimated Duration
- Completion Checklist

---

# Success Criteria

A successful task breakdown:

- Covers the complete project scope.
- Is logically organized.
- Has clear ownership.
- Defines dependencies.
- Supports project tracking.
- Enables accurate scheduling.
- Reduces execution ambiguity.
- Improves accountability.
